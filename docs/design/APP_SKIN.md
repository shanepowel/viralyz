# App skin reference

These two screens are the **reference implementation** for the product UI. Every other screen reuses these components:

- Panel
- Stat card
- Score ring at **34px** (table) and **150px** (hero)
- Status chip
- Fix card (pending + applied with earned delta)
- Sticky action bar
- Next best action (NBA) card
- App shell sidebar (Create / Grow / Earn journey map)

## Source mockup

Open `docs/design/app-skin-mockup.html` in a browser. Toggle **Dashboard** and **Score results** in the bottom right.

## UX principles

1. One primary action always visible: violet **Score content** in the sidebar.
2. Sidebar is a journey map (Create / Grow / Earn), not a feature list.
3. Exactly four things above the fold on Home: monthly score, prediction accuracy, Next Best Action (gradient, sole loud CTA).
4. Score colour is the only loud colour; warm neutrals everywhere else.
5. Results speak in plain sentences. Fix cards show applied trail ("Applied, earned +9").
6. Same warm paper, Bricolage Grotesque, violet, and rings as the marketing site.
7. Zero em dashes in product copy.
